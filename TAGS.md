# Tagging Convention

Use the first comment line in each problem file for searchable tags:

```python
# Tags: two-pointers, binary-search
```

Guidelines:

- Keep tags lowercase.
- Use hyphens for multi-word tags.
- Put the dominant pattern first.
- Add as many secondary tags as needed.
- Keep all problem files at the repository root.

Examples:

- `hash-map`
- `two-pointers, sorting`
- `binary-search, rotated-array`

See also: `README.md` for a cross-reference index.
